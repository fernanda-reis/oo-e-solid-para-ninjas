Em um sistema orientado a objetos, a implementação importa, mas pensar no funcionamento e interação das classes entre si é fundamental. Cada classe é como um quebra-cabeças que, se alterado, propagará alterações a todas as peças ao redor, criando a necessidade de ajustes. Na orientação a objetos, tentamos criar estratégias para que essas alterações sejam as mais simples possíveis, facilitando manutenção e escalabilidade do projeto.
